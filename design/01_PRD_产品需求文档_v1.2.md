# MaxLuLu AI 产品需求文档（PRD）

> 版本：v1.1 | 日期：2026-05-01 本文档是 MaxLuLu AI 全站开发的唯一产品真相源。所有功能开发必须对照本文档。

---

## 一、产品定位

**产品名称：** MaxLuLu AI

**一句话定位：** AI 驱动的高端针织印花连衣裙设计 × 社交电商平台

**品牌 Slogan：** Fashion For You — 每一朵印花，都由你绽放

**品牌背景：** 钱晓禄 2009 年上海创立，15 年历史，核心产品为针织印花连衣裙（对标 DVF 裹身裙）。2026 年转型为 AI 设计 + 社交电商。

**法律主体：** 上海禄创企业管理有限公司（91310107MA7AHBHY8R）

**核心差异化：**

- 不是卖现成衣服，是"AI 设计 + 众定制造"
- 消费者参与设计过程（选印花、选款式、选面料）
- 30 人参团模式降低定制门槛
- 设计师零门槛入驻，AI 工具赋能

---

## 二、用户角色

### 2.1 消费者（C 端，核心用户）

**画像：** 25-45 岁女性，关注时尚，喜欢独特印花，愿意为设计感付费。

**核心需求：**

- 浏览有设计感的印花连衣裙
- 以合理价格买到（参团 ¥399-999，比定制便宜）
- 或定制独一无二的款式（¥1599-2999）
- 邀请朋友参团，双方获 ¥30 优惠

**关键路径：**

```
首页 → 印花衣橱浏览 → 选中设计 → 查看详情 → 选择参团或定制 → 支付 → 等待发货
```

**转化目标：** 参团下单 / 定制下单

### 2.2 设计师（B 端）

**画像：** 服装设计专业学生或独立设计师，有创意但缺少供应链和销售渠道。

**核心需求：**

- 用 AI 工具快速生成设计
- 发布到平台让消费者购买
- 赚取 10% 分润（满 ¥100 提现）

**关键路径：**

```
了解平台 → 申请入驻 → 审核通过 → 使用 AI 工具创作 → 发布设计 → 消费者购买 → 收到分润
```

**转化目标：** 入驻申请 → 发布第一个设计

### 2.3 创作型消费者

**画像：** 对设计感兴趣的消费者，想自己参与设计过程。

**核心需求：**

- 用 AI 工具生成自己喜欢的印花/款式
- 定制 1 件自己的设计
- 也可以发布到平台让别人参团买

**关键路径：**

```
AI 工具体验 → 生成设计 → 选择定制自穿 或 发布平台
```

### 2.4 平台管理员

**画像：** 品牌运营人员（当前为创始人本人）。

**核心需求：**

- 审核设计师入驻申请
- 审核设计师发布的作品
- 管理拼团（手动 fulfill/expire/extend）
- 管理提现申请

## 四、业务规则

### 4.1 参团规则

|规则|值|
|---|---|
|成团人数|30 人|
|有效期|72 小时（从建团开始）|
|满团处理|立即标记 fulfilled，进入生产|
|未满处理|自动全额退款 + 邀请码失效|
|到期检查|Cron 每日执行（Hobby 计划限制）|
|消费者可见|当前已参团人数 / 30、剩余时间倒计时|

### 4.2 定价体系

|面料档位|参团价（30件）|定制价（1件）|
|---|---|---|
|基础|¥399 - ¥599|¥1,599 - ¥1,999|
|中端|¥599 - ¥799|¥1,999 - ¥2,599|
|高端|¥799 - ¥999|¥2,599 - ¥2,999|

### 4.3 邀约裂变

|规则|值|
|---|---|
|优惠金额|邀请人和被邀人各 ¥30|
|抵扣方式|直接体现在订单 discountAmount|
|触发时机|被邀人参团下单时，事务内校验邀请码有效性|
|限制|每个邀请码只能被使用一次|

### 4.4 设计师分润

|规则|值|
|---|---|
|分润比例|10%（全站统一，以本文档为准）|
|结算时机|消费者确认收货后 7 天|
|提现门槛|¥100|
|提现方式|支付宝|

### 4.5 支付

|规则|值|
|---|---|
|支付方式|支付宝（手机网站 + 电脑网站）|
|手续费|0.6%|
|订单超时|30 分钟未支付自动关闭|
|金额校验|后端查库算金额，不接受前端传金额|
|金额存储|整数分（如 59900 = ¥599.00）|

### 5.2 关键字段约束

- **金额**：所有金额字段用整数分（Int），展示时 /100 转元
- **订单状态流转**：见《User Flow v1.1 → 第九章 订单状态机》

- **拼团状态流转**：active → fulfilled / expired / cancelled
- **设计审核状态**：pending → approved / rejected

## 一、新增产品定位说明

MaxLuLu AI 从"AI 服装设计灵感画廊 × 参团采购 × 个人定制 × 设计师创业平台"
扩展为消费者也可以创作 + 分享 + 商业化的 UGC 平台。

**新核心循环**:消费者创作 → 分享灵感广场 → 获得反馈 → 商业化(被买/被团购)→ 升级为设计师

## 二、新页面定义

### 2.1 我的设计工作室 /my-studio

**面向**:消费者(认证设计师走 /studio,功能更全)
**目的**:让普通消费者也能创作专属服装设计

**4 种创作入口**:

1. **基于喜欢的款式做相似印花** —— 从印花衣橱/灵感广场点"基于这个改造"进入,带入原作品参数
2. **从零设计印花** —— 空白画布开始
3. **基于个人需求生成花型** —— 输入场景(职场/约会/休闲)、性格关键词、身材比例,AI 推荐花型
4. **上传自己的图片做素材** —— 上传图片作为印花生成的灵感源

**给消费者的 AI 工具**(中等版):

- 印花生成
- 四方连续
- 图案上身
- AI 试衣

**3 个出口**:

1. 单独下单定制(¥1599-2999)
2. 发起团购(30 件起)
3. 分享到灵感广场(可不消费,纯分享)

**创作次数**:5 次/天(沿用 PRD v1.1 §6.4)

### 2.2 灵感广场 /inspiration

**面向**:所有人(公开,游客也能浏览)
**目的**:展示创作过程,激发设计灵感,促进商业化

**展示内容**:

- 创作效果图(必显)
- 创作者信息(必显)
  - 认证设计师:卡片金色描边 + 右上角金色角标
  - 普通用户:无特殊视觉标识
- prompt + 参数:**创作者自选**
  - 选项 A:免费公开
  - 选项 B:付费/积分解锁查看
  - 选项 C:完全不公开(只展示效果图)

**排序**:推荐算法 + 用户可切换排序

- 默认:推荐(综合点赞、收藏、团购量、浏览量、时效衰减)
- 备选:最新、最热、价格升降序、AI 工具类型筛选

**消费者动作**:

- 点赞 / 收藏 / 评论
- 解锁 prompt(付费或用积分)
- "基于这个改造" → 跳 /my-studio 携带参数
- "发起团购" → 提议团购(作者收到通知,同意后才能开团)
- "我也要这个" → 催促作者上架

### 2.3 灵感作品详情页 /inspiration/[id]

**面向**:所有人
**展示内容**:大图、作品故事、创作者信息、prompt 解锁状态、参数详情(已解锁时)、相似作品推荐
**操作**:点赞、收藏、评论、解锁 prompt、基于这个改造、发起团购

## 四、收入分润(替换 PRD v1.1 §8)

### 4.1 阶梯式分润比例

**普通用户作品**(累计销售件数计):

| 累计销售件数 | 创作者分润 |
|---|---|
| 0 - 100 件 | 5% |
| 101 - 500 件 | 7% |
| 501 - 1000 件 | 9% |
| 1001 件以上 | 10% |

**认证设计师作品**:

| 累计销售件数 | 设计师分润 |
|---|---|
| 0 - 100 件 | 10% |
| 101 - 500 件 | 13% |
| 501 - 1000 件 | 16% |
| 1001 件以上 | 20% |

### 4.2 分润规则

- 分润基数 = 实付金额(扣除优惠券、邀约直减后)
- 工厂生产成本 + 平台运营成本由平台承担,从其余毛利中支付分润
- 退款后分润回收
- 结算周期:确认收货后 7 天
- 提现门槛:满 ¥100 支付宝提现
- 阶梯阈值按"累计已售件数"计算,不重置

### 4.3 prompt 解锁收入分润

- 创作者自定义解锁价格(范围 ¥1 - ¥30 / 次)
- 平台抽成 30%,创作者拿 70%
- 积分解锁:用户消耗积分换取查看权,创作者按比例获得积分回报(具体规则待定)

## 六、用户升级路径(新增)

### 6.1 普通用户 → 认证设计师

**自动升级邀请触发条件**(满足任一):

- 累计作品赞数 ≥ 500
- 累计作品收藏数 ≥ 200
- 累计作品被购/团购件数 ≥ 30

**触发后**:

1. 系统发站内消息 + 短信邀请
2. 用户进入 /studio/join 走入驻流程(填补充信息、签协议)
3. 管理员审核(7 个工作日内)
4. 通过 → 升级为认证设计师,作品历史保留

**主动申请**:

- 用户也可不等邀请,直接走 /studio/join 主动申请
