## 01. 全局设计系统摘要

### 1.1 品牌色彩系统

本次高保真稿采用 **Porcelain Cyan × Smoked Rose / 瓷青蓝 × 烟粉玫瑰** 作为主 UI 配色。

整体气质应保持：

- 高端女装品牌感
- 轻盈、柔和、知性
- 适合针织印花连衣裙、A 字裙、深 V 裹身裙
- 不使用大面积金色和米色
- 不使用纯黑白大反差
- 金色仅用于「认证设计师」身份系统，不作为品牌主色

## 02. 认证设计师视觉一致性

认证设计师系统必须在以下四处保持一致：

1. 灵感广场卡片
2. 印花衣橱商品卡片
3. 详情页作者署名区域
4. 评论区头像

### 2.1 认证设计师核心样式

| 元素 | 样式 |
|---|---|
| 认证主色 | `#C8A875` |
| 认证描边 | `1.5px solid #C8A875` |
| 认证角标背景 | `#C8A875` |
| 认证角标文字 | `#FFFFFF` |
| 认证角标字号 | `12px / 16px` |
| 认证徽章圆角 | `999px` 或角标斜切 |
| 认证徽章图标 | 推荐使用 crown / shield / verified mark |
| 认证徽章阴影 | `0 2px 6px rgba(200, 168, 117, 0.24)` |

### 2.2 灵感广场卡片认证样式

| 状态 | 样式 |
|---|---|
| 普通作品卡片 | `border: 1px solid #D2DEDF` |
| 认证设计师作品卡片 | `border: 1.5px solid #C8A875` |
| 认证角标位置 | 右上角，覆盖在图片区域上 |
| 认证角标尺寸 | 高 `24px`，内边距 `0 8px` |
| 认证角标文字 | `认证` |
| 认证卡片 Hover | 保留金色描边，阴影增强 |
| 认证卡片禁用 | 不降低认证徽章透明度，仅降低内容透明度 |

### 2.3 印花衣橱商品卡片认证样式

| 元素 | 样式 |
|---|---|
| 卡片描边 | `1.5px solid #C8A875` |
| 右上角角标 | 金色底，白色文字 `认证` |
| 作者署名 | `by [名字]` |
| 作者名旁徽章 | 金色小圆章，尺寸 `14px` |
| 商品价格 | 不因认证改变颜色 |
| 普通用户作品 | 不显示描边，不显示角标 |

### 2.4 详情页作者署名认证样式

| 元素 | 样式 |
|---|---|
| 作者信息卡 | `border: 1px solid #D2DEDF` |
| 认证作者信息卡 | `border: 1px solid #C8A875` |
| 头像尺寸 Desktop | `40px × 40px` |
| 头像尺寸 Mobile | `36px × 36px` |
| 作者名 | Text Primary |
| 认证 Badge | 金色浅底或金色描边，文字 `认证设计师` |
| 查看更多作品按钮 | Secondary Button，跳转作者主页占位链接 |

### 2.5 评论区认证头像样式

| 元素 | 样式 |
|---|---|
| 评论头像 | 圆形头像 |
| 认证标识 | 头像右下角小金章 |
| 小金章尺寸 | `12px × 12px` |
| 小金章边框 | `1px solid #FFFFFF` |
| 悬停说明 | Tooltip：`认证设计师` |

### 2.6 认证系统开发规则

- 认证视觉必须由统一字段控制，例如：`isVerifiedDesigner: boolean`
- 不允许各页面单独写死认证样式
- 认证金 `#C8A875` 只用于身份识别，不用于普通促销
- 移动端认证角标不可隐藏
- 图片加载失败时，认证徽章仍需保留
- 列表卡片、详情页、评论区必须复用同一套认证组件

## 03. 数字规范

所有数字、计数、价格必须使用等宽数字，并保持衬线对齐。

### 3.1 数字字体规则

| 场景 | 字体 | CSS |
|---|---|---|
| 价格 | Playfair Display | `font-variant-numeric: tabular-nums;` |
| 统计数字 | Playfair Display | `font-feature-settings: "tnum" 1;` |
| 评论数 / 点赞数 | Inter | `font-variant-numeric: tabular-nums;` |
| 参数数字 | Inter | `font-variant-numeric: tabular-nums;` |
| 表格数字 | Inter | `font-variant-numeric: tabular-nums;` |

### 3.2 推荐 CSS

```css
.number,
.price,
.count,
.metric {
  font-variant-numeric: tabular-nums;
  font-feature-settings: "tnum" 1, "lnum" 1;
}

.price {
  font-family: "Playfair Display", "Noto Serif SC", serif;
  letter-spacing: 0.01em;
}

.metric {
  font-family: "Playfair Display", "Inter", sans-serif;
}
````

### 3.3 价格格式

|场景|格式|示例|
|---|---|---|
|商品卡价格|`¥389`|`¥389`|
|解锁价格|`¥39 解锁`|`¥39 解锁`|
|详情页价格|`¥ 389`|`¥ 389`|
|原价|删除线 + Tertiary|`¥699`|
|会员价|Accent Subtle 背景|`会员价 ¥349`|
|区间价格|`¥1 - ¥29`|`¥1 - ¥29`|

### 3.4 计数格式

|数字大小|格式|
|---|---|
|0 - 999|直接显示|
|1,000 - 999,999|千分位逗号|
|1,000,000 以上|可使用中文缩写，但后台保留原始值|
|金额|不使用中文缩写|
|点赞 / 收藏|可使用 `1.2k`，但详情页展示完整数字|

### 3.5 对齐规则

|场景|对齐|
|---|---|
|Hero 统计数字|居中或左对齐，多个数字基线一致|
|商品卡价格|左对齐|
|详情页价格|左对齐，货币符号与数字基线一致|
|参数表格|数字右对齐|
|数据指标|图标居中，数字居中|
|解锁价格按钮|居中对齐|

### 4.5 移动端行为

- 筛选侧栏改为底部抽屉 Bottom Sheet
    
- 右侧信息区改为页面下方内容
    
- CTA 固定在底部安全区上方
    
- Hover 操作改为点击 / 长按 / 更多菜单
    
- 商品卡保持 2 列时，文字最多 2 行省略
    
- 详情页主图比例保持稳定，避免 CLS 抖动
    
- 底部导航高度建议 `64px + safe-area-inset-bottom`

## 08. 实施注意事项

### 8.1 瀑布流性能优化

- 使用 Masonry CSS Grid 或 Masonry 组件实现
    
- 图片必须懒加载
    
- 推荐使用 `IntersectionObserver`
    
- 图片加载前使用骨架屏或占位色
    
- 必须预设图片宽高或 aspect-ratio，避免 CLS
    
- 长列表必须分页或无限加载
    
- 无限加载必须显示 loading 状态
    
- 图片失败时显示默认占位图
    

推荐图片比例：

|场景|比例|
|---|---|
|商品卡|`4:5`|
|灵感卡|`3:4` / `4:5`|
|详情页主图|`4:5` 或自适应|
|缩略图|`1:1`|
|横向推荐图|`4:5`|

### 8.2 筛选栏与 Sticky 行为

- Desktop 右侧筛选栏使用 sticky
    
- Sticky 顶部距离应避开 Header
    
- 推荐：`top: 88px`
    
- 筛选栏高度超过视口时允许内部滚动
    
- 不要让整个页面出现双滚动条
    
- 移动端筛选栏必须改为 Bottom Sheet
    
- Bottom Sheet 关闭后保留筛选状态
    

### 8.3 详情页移动端 CTA

- 移动端详情页 CTA 必须固定底部
    
- 避免被 iOS Home Indicator 遮挡
    
- 使用 `safe-area-inset-bottom`
    
- 主按钮优先级最高
    
- 次级按钮可并排展示
    
- 当用户滚动到页面底部，CTA 不应遮挡评论输入框
    
- 键盘弹出时，底部 CTA 应隐藏或上移
    

推荐 CSS：

```css
.mobile-sticky-cta {
  position: sticky;
  bottom: 0;
  padding: 12px 16px calc(12px + env(safe-area-inset-bottom));
  background: #FFFFFF;
  border-top: 1px solid #E1E8E8;
  z-index: 50;
}
```

### 8.4 交互与设备兼容

- Desktop 使用 Hover
    
- Mobile 使用点击、长按、更多菜单
    
- 所有 Hover 操作必须有移动端替代方案
    
- 按钮点击区域不小于 `44px`
    
- 输入框高度不小于 `40px`
    
- 移动端底部导航高度建议 `64px`
    
- 所有弹窗必须支持 ESC 关闭
    
- 所有抽屉必须支持点击遮罩关闭
    
- 图片全屏查看必须支持左右滑动
    

### 8.5 组件复用原则

以下组件必须封装复用，不允许页面内重复实现：

|组件|说明|
|---|---|
|`VerifiedBadge`|认证设计师徽章|
|`CertifiedCardFrame`|认证卡片描边与角标|
|`AuthorSignature`|作者署名与认证标识|
|`PromptVisibilityBlock`|Prompt 免费 / 付费 / 不公开状态|
|`PriceText`|价格展示与数字对齐|
|`MetricItem`|点赞 / 收藏 / 购买数据|
|`StatusBadge`|作品状态标签|
|`BottomActionBar`|移动端底部 CTA|
|`FilterDrawer`|移动端筛选抽屉|
|`MasonryGrid`|瀑布流网格|

### 8.6 数据字段与接口建议

|字段|类型|说明|
|---|---|---|
|`creatorType`|string|`designer` / `user` / `brand`|
|`isVerifiedDesigner`|boolean|是否认证设计师|
|`promptVisibility`|string|`free` / `paid` / `private`|
|`unlockPrice`|number|解锁价格|
|`workStatus`|string|`draft` / `shared` / `listed` / `sold`|
|`toolType`|string|`print` / `repeat` / `tryon` / `fitting`|
|`metrics.likes`|number|点赞数|
|`metrics.favorites`|number|收藏数|
|`metrics.orders`|number|被购数|
|`createdAt`|string|创建时间|
|`updatedAt`|string|更新时间|
|`authorProfileUrl`|string|作者主页链接，P2 可占位|
|`remainingQuota`|number|当日剩余创作次数|
|`upgradeProgress`|number|认证设计师升级进度|

### 8.7 可访问性与对比度

- 正文与背景对比度建议不低于 `4.5:1`
    
- 主要按钮文字与背景对比度不低于 `4.5:1`
    
- 禁用态不只依赖颜色区分
    
- 认证状态不只依赖金色，应同时显示文字 `认证`
    
- Prompt 锁定状态不只依赖模糊，应同时显示锁图标与文案
    
- 所有图标按钮必须有 `aria-label`
    
- 图片必须有 `alt`
    
- 表单输入必须有 label 或 aria-label
    
- 键盘用户必须可以访问：
    
    - 筛选项
        
    - Tab
        
    - 卡片快捷操作
        
    - 解锁按钮
        
    - 评论提交
        
- 焦点态统一使用：
    

```css
:focus-visible {
  outline: 2px solid #2F5A5D;
  outline-offset: 2px;
}
```

### 8.8 视觉还原注意事项

- 高保真稿中的图片以实际商品图替换时，必须保持统一比例
    
- 商品图不要拉伸变形
    
- 卡片高度允许因文字不同略有变化，但按钮与价格区尽量对齐
    
- 页面背景不得改成纯白大面积铺满
    
- 不要把 `#C8A875` 扩展为促销色或普通强调色
    
- 不要把 Smoked Rose 用成大面积背景
    
- 主要操作按钮数量要克制，避免淘宝化
    
- 所有页面的认证视觉必须完全一致
    
- 移动端优先保证购买、解锁、改造三个关键动作可见
    
- WebP 优先，JPG/PNG 作为降级
    
- 图片建议上传前压缩，单张首屏图建议小于 `300KB`
    
- 首屏大图应使用响应式图片：
    
    - Desktop：`1440px`
        
    - Tablet：`1024px`
        
    - Mobile：`750px`
