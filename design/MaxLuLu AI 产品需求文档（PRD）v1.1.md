### 5.2 关键字段约束（v1.1 更新）

- **金额**：所有金额字段用整数分（Int），展示时 /100 转元
- **订单状态流转**：见《User Flow v1.1 → 第九章 订单状态机》
    - 定制订单：pending_payment → paid → in_production → shipped → completed
    - 参团订单：pending_payment → paid → grouping → group_success → in_production → shipped → completed
    - 异常路径：closed（超时）、refund_pending → refunded（退款）、group_failed → refunded
- **拼团状态流转**：active → fulfilled / expired / cancelled
- **设计审核状态**：pending → approved / rejected

### 6.3 路由保护规则

|路由|保护方式|
|---|---|
|/ , /products, /products/[id]|公开，无需登录|
|/studio/join|公开，无需登录（公开布局）|
|/studio (工具首页)|公开预览，生成功能需登录|
|/products/[id]/custom, /group-buy/*|操作时弹登录框|
|/my|需登录，未登录重定向到首页|
|/studio/pattern/_, /studio/fashion/_|工具预览公开，点"生成"时检查登录 + 次数限制|
|/studio/publish|需 designer 角色|
|/studio/dashboard|需 designer 或 designer_pending 角色|
|/admin/manage|需 admin 角色|

### 6.4 AI 工具使用次数限制

|角色|每日生成次数|说明|
|---|---|---|
|guest|0|只能预览工具界面|
|consumer|5 次/天|体验后引导入驻或购买|
|designer_pending|20 次/天|审核期间可充分体验|
|designer|不限|正式设计师无限制|
|admin|不限||

> ⚠️ 次数限制为 MVP 初始值，后续可调整或改为积分制。
